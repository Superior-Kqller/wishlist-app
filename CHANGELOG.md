# Changelog

Все значимые изменения документируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
проект использует [Semantic Versioning](https://semver.org/lang/ru/).

## [1.6.2] — 2026-03-20

### Добавлено

- **Автопроверка Prisma-миграций** — новый скрипт `db:check-migrations` (`scripts/check-prisma-migrations.mjs`) блокирует изменения `prisma/schema.prisma` без соответствующего `prisma/migrations/*/migration.sql`

### Изменено

- **CI (Docker Image workflow)** — добавлен шаг проверки миграций на `push` и `pull_request`, чтобы релизы не уходили с несинхронизированной схемой БД

## [1.6.1] — 2026-03-20

### Исправлено

- **Фильтры пользователей** — убрано хрупкое переключение через скрытый tab-state; выбор режимов `Все/Мои/Пользователь` стал стабильным и предсказуемым
- **Невалидный `userId` в URL** — добавлена нормализация с мягким fallback в режим «все», чтобы не оставаться в пустом/сломанных состоянии фильтрации
- **Подборки в фильтрах** — единая логика user-scope для desktop и mobile, включая корректный сброс зависимой подборки при смене контекста

### Улучшено

- **Safari iOS UI** — увеличены tap-target до мобильных best practices, улучшены параметры viewport (`svh/dvh`) и nested scroll для drawer/select/dialog
- **Select и Tabs интеракции** — более спокойные и консистентные состояния (без резких scale-эффектов), что улучшает визуальную цельность интерфейса
- **UI-устойчивость** — унифицирована логика вычисления user-scope в отдельной утилите `src/lib/filter-state.ts`

## [1.6.0] — 2026-03-19

### Добавлено

- **Удаление своих комментариев** — `DELETE /api/items/[id]/comments/[commentId]`, кнопка в модалке товара
- **`src/lib/user-pagination.ts`** — кодирование курсора списка пользователей (админка)
- **`ensureUserIdsExist`** (`list-utils`) — проверка существования пользователей по ID

### Исправлено

- **Модалка товара** — кнопки действий в один ряд с горизонтальным скроллом на узкой ширине (без переноса «Удалить»)
- **Фильтры** — при смене пользователя сбрасывается подборка в URL; мобильный drawer показывает те же подборки, что и десктоп; недопустимый `listId` для текущего режима сбрасывается автоматически

### Улучшено

- **API** — `getSessionUserIdVerified`: ID из сессии только если пользователь есть в БД; единообразные ответы 401 на ряде маршрутов
- **GET /api/tags** — в выдаче только теги с товаров в подборках, доступных текущему пользователю (приватный кэш)
- **Админка: GET /api/users** — устойчивая cursor-пагинация по `createdAt` + `id`, проверка параметра `limit`
- **POST /api/lists** — валидация `viewerIds`: отклонение запроса с несуществующими пользователями
- **Карточка товара (сетка)** — доступность: `role`, фокус-кольцо, активация по Enter/Space, `aria-label` для открытия деталей / выбора

### Изменено

- Общая утилита `filterListsBySelectedUser` (`src/lib/list-filter-client.ts`) для `CombinedFilter` и `FiltersDrawer`
- Ряд handlers API переведён на `getSessionUserIdVerified` и уточнённые ответы об ошибках

## [1.0.0] — 2026-03-17

Первый релиз.

### Основные возможности

- **Аутентификация** — NextAuth.js с JWT-сессиями, ролями USER/ADMIN
- **Вишлист** — CRUD для желаний с изображениями, ценами, приоритетами (1-5 звёзд), тегами, заметками
- **Подборки** — создание списков с шарингом между пользователями
- **Парсер URL** — автоматическое извлечение данных товаров с Wildberries (API), Ozon, AliExpress, generic-сайтов
- **Серверный поиск** — ILIKE по названию, заметкам и тегам с debounce 300ms
- **Бесконечная прокрутка** — cursor-пагинация + Intersection Observer
- **Фильтры в URL** — search, sort, tags, purchased синхронизируются с query params
- **Быстрый приоритет** — hover-превью и клик по звёздам прямо на карточке
- **Bulk-операции** — режим выделения, массовое удаление и отметка «куплено»
- **Экспорт** — CSV и JSON через /api/items/export
- **Темы** — 7 цветовых тем + светлый/тёмный/системный режим
- **Адаптивный дизайн** — мобильные фильтры, touch-свайп галереи
- **Администрирование** — управление пользователями, сброс паролей
- **Настройки профиля** — аватар, имя, смена пароля
- **Комментарии** — к каждому желанию
- **Статистика** — дашборд с ценами по валютам

### Безопасность

- SSRF-защита парсера — DNS resolve + проверка приватных IP-диапазонов
- Rate limiting через Valkey (Unix socket) с in-memory fallback
- CSP nonce для inline-скриптов
- Bcrypt (12 rounds), Zod-валидация, защита от удаления последнего админа

### Инфраструктура

- Docker multi-stage build, GitHub Actions → GHCR
- PostgreSQL 17 + Prisma 7 (driver adapters)
- Valkey 8.1 для rate limiting (Unix socket)
- Автоматический seed пользователей при первом запуске
