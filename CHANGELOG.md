## [Unreleased]

### Добавлено

- **Удаление своих подборок** — в диалоге редактирования подборки кнопка «Удалить подборку» и подтверждение; `DELETE /api/lists/[id]` (уже был): желания остаются без привязки к подборке; при активном фильтре по удалённой подборке сбрасывается `listId` в URL

### Изменено

- **Карточка желания (модалка)** — на мобильных почти на всю ширину/высоту экрана (`94dvh`), ниже блок фото (`max 42vh`), действия столбцом на всю ширину без горизонтального скролла, кнопки не ниже 44px; кнопка «Закрыть» во всех диалогах — круг 44×44, выше слоя галереи (`z-[60]`)
- **UI главной** — более узкая липкая полоса фильтров; сортировка, «глаз» и «Выбрать» перенесены в один ряд с пользователем/подборками; поиск на отдельной строке; больше glassmorphism у карточек и «Добавить товар»; на десктопе фон карточки плотнее (`sm:`); цена приглушена (`muted` + лёгкий violet); единые варианты кнопок `glass` / `glassActive`, смягчён `outline`; `filterBarTriggerClass` вынесен в `src/lib/filter-toolbar-styles.ts`
- **Главная / фильтр подборок** — по умолчанию снова «все подборки» без `listId` в URL; устаревший `listId=all` при открытии убирается из адреса
- **Добавление желаний** — кнопки и «+» на сетке не зависят от фильтра; в форме создания подборка **обязательна**, в списке выбора только **свои** подборки, по умолчанию подставляется **первая своя** (по названию)
- **GET /api/items** — `listId=all` (legacy) не фильтрует по подборке
- **Тултипы** (`TooltipContent`) — приглушённый фон `popover`, тонкая рамка, лёгкий blur и тень; убран «прыгающий» zoom; сдвиг появления уменьшен

- **PWA** — `@ducanh2912/next-pwa`, `src/app/manifest.ts`, `appleWebApp` в layout; исключения в `proxy.ts` для `sw.js` / `workbox-*.js` / `manifest.webmanifest`
- **`src/app/global-error.tsx`** — автономная страница ошибки без провайдеров (корректный пререндер)

### Изменено

- **Новый brandbook (glassmorphism v2)** — обновлены dark-токены и glass-слой в `src/app/globals.css`: фон/поверхности/бордеры, blur и свечение приведены к новому визуальному языку
- **Компоненты интерфейса** — `Button`, `Card`, `Input`, `Select`, `Badge`, `FiltersDrawer` и блоки фильтров синхронизированы со стеклянной стилистикой и единым ритмом размеров
- **Хедер и логотип** — увеличен lockup-логотип, обновлены высота шапки и размеры мобильных action-кнопок для лучшей читаемости
- **Mobile toolbar** — скорректированы sticky-offset и плотность элементов на главной странице для более аккуратного поведения на телефонах

### Исправлено

- **Парсер ссылок — длинные цепочки редиректов** — перед загрузкой HTML и API маркетплейсов URL канонизируется через `HEAD` + `redirect: "manual"` (до 12 шагов, SSRF на каждом шаге): короткие ссылки вроде Ozon `/t/...` стабильнее разворачиваются в финальный адрес; дублирующий `GET`-разворот только для `/t/` внутри `parseOzon` убран
- **Desktop фильтр пользователя** — устранён конфликт двойного обновления URL в `CombinedFilter`, из-за которого выбор пользователя визуально срабатывал, но не применялся
- **Единообразие выбора пользователя** — обработка пунктов в desktop-dropdown приведена к тому же событийному паттерну, что и на mobile, чтобы исключить платформенные расхождения
- **Rate limit авторизации (credentials)** — добавлен server-side pre-check для `POST /api/auth/callback/credentials` с возвратом `429` при превышении лимита
- **Avatar SSRF hardening** — введена строгая политика `avatarUrl` (локальные `/uploads/...` или внешние `https` только из `AVATAR_ALLOWED_HOSTS`), убран wildcard в `next/image` remote patterns
- **Смена пароля self-service** — для смены собственного пароля теперь обязателен `currentPassword` (включая admin self-change), неверный/отсутствующий пароль возвращает `400`
- **Seed safety в production** — seed блокирует дефолтные/пустые пароли и невалидные логины пользователей, добавлен fail-fast с понятной ошибкой
- **SSRF deny-list парсера** — расширены блокировки внутренних диапазонов (`100.64.0.0/10`, `198.18.0.0/15`)
- **pg DeprecationWarning в Docker** — в `package.json` зафиксирован `pg@8.18.0`: в `pg` ≥8.19 предупреждение о параллельных `client.query()` на одном соединении; `@prisma/adapter-pg` в отдельных сценариях это всё ещё провоцирует. После исправления в Prisma можно снова поднять версию `pg`

## [1.0.0] — 2026-03-17

Первый релиз.

### Основные возможности

- **Аутентификация** — NextAuth.js с JWT-сессиями, ролями USER/ADMIN
- **Вишлист** — CRUD для желаний с изображениями, ценами, приоритетами (1-5 звёзд), тегами, заметками
- **Подборки** — создание списков с шарингом между пользователями
- **Парсер URL** — автоматическое извлечение данных товаров с Wildberries (API), Ozon, AliExpress, generic-сайтов
- **Серверный поиск** — ILIKE по названию, заметкам и тегам с debounce 300ms
- **Бесконечная прокрутка** — cursor-пагинация + Intersection Observer
- **Фильтры в URL** — search, sort, tags, purchased синхронизируются с query params
- **Быстрый приоритет** — hover-превью и клик по звёздам прямо на карточке
- **Bulk-операции** — режим выделения, массовое удаление и отметка «куплено»
- **Экспорт** — CSV и JSON через /api/items/export
- **Темы** — 7 цветовых тем + светлый/тёмный/системный режим
- **Адаптивный дизайн** — мобильные фильтры, touch-свайп галереи
- **Администрирование** — управление пользователями, сброс паролей
- **Настройки профиля** — аватар, имя, смена пароля
- **Комментарии** — к каждому желанию
- **Статистика** — дашборд с ценами по валютам

### Безопасность

- SSRF-защита парсера — DNS resolve + проверка приватных IP-диапазонов
- Rate limiting через Valkey (Unix socket) с in-memory fallback
- CSP nonce для inline-скриптов
- Bcrypt (12 rounds), Zod-валидация, защита от удаления последнего админа

### Инфраструктура

- Docker multi-stage build, GitHub Actions → GHCR
- PostgreSQL 17 + Prisma 7 (driver adapters)
- Valkey 8.1 для rate limiting (Unix socket)
- Автоматический seed пользователей при первом запуске
